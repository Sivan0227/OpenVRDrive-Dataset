# Trial Assignment Information

## Overview

This folder contains detailed information about the experimental design and trial assignments for Scenario 1. Due to the sampling strategy employed to prevent participant fatigue, each of the 60 participants completed only 16 out of the 22 available sub-scenarios. This document provides the complete assignment matrix showing which specific scenarios each participant experienced.

## Scenario Assignment Matrix

The table below shows the exact scenario combinations completed by each participant. Each row represents one participant (identified by `person_id`), and each column represents one sub-scenario. 

### Reading the Table

- **✓ (checkmark)**: Indicates the participant completed this specific scenario
- **nan (or blank)**: Indicates the participant did not experience this scenario
- **Column Headers**: Format is `[Lane-Change Type]`<br>`[Traffic Type]`<br>`[Scenario ID]`
  - Lane-Change Type: DLC (Discretionary) or MLC (Mandatory)
  - Traffic Type: Mixed (CAV platoon) or Traditional (HDV platoon)
  - Scenario ID: Unique identifier (0-13 for DLC, 0-7 for MLC)

### Assignment Strategy

As described in the main scenario documentation:

**Fixed Scenarios (All Participants):**
- All participants completed **all 4 traditional traffic scenarios**:
  - DLC Traditional: Scenarios 12, 13
  - MLC Traditional: Scenarios 6, 7

**Sampled Scenarios (Counterbalanced):**
- Each participant completed **8 out of 12** DLC mixed traffic scenarios (scenarios 0-11)
- Each participant completed **4 out of 6** MLC mixed traffic scenarios (scenarios 0-5)

**Total per Participant:** 16 scenarios (4 traditional + 8 DLC mixed + 4 MLC mixed)

### Counterbalancing Design

The assignment follows a Latin square design to ensure:
1. **Balanced Coverage**: Each sub-scenario is experienced by approximately 40 participants (2/3 of 60)
2. **Minimized Order Effects**: Scenarios are randomized across participants
3. **Controlled Fatigue**: Session duration kept to ~45-65 minutes per participant
4. **Statistical Power**: Sufficient sample size for each condition (N≈40 per sub-scenario)

## Additional Files in This Folder

### scenario1_trial_params.json
JSON configuration file containing the complete parameter specifications for all 22 sub-scenarios. This file is used by the data processing tools in `/data_processing_tools/` to retrieve scenario parameters programmatically.

**Structure:**
- `scenario_name`: Scenario description
- `scenario_id`: Unique identifier (1 for this scenario)
- `parameter_names`: Parameter column definitions
  - `discretionary`: Parameters for DLC scenarios (cav_max_speed, cav_time_gap, front_hdv_speed, platoon_size)
  - `mandatory`: Parameters for MLC scenarios (cav_max_speed, cav_time_gap, platoon_size)
- `discretionary_params`: 14 parameter combinations for DLC mixed/traditional scenarios
- `mandatory_params`: 8 parameter combinations for MLC mixed/traditional scenarios

**Example Usage:**
```python
import json
from pathlib import Path

# Load scenario parameters
config_path = Path('scenario1_trial_params.json')
with open(config_path, 'r') as f:
    config = json.load(f)

# Access DLC scenario parameters
dlc_params = config['discretionary_params']  # Returns list of [speed, time_gap, front_hdv_speed, platoon_size]

# Access MLC scenario parameters
mlc_params = config['mandatory_params']  # Returns list of [speed, time_gap, platoon_size]
```

## Usage Notes

**For Data Analysis:**
- Use this assignment matrix to identify which participants contributed data to each condition
- Account for missing data patterns in statistical models (each mixed scenario has ~40 participants, not 60)
- Consider participant ID as a random effect in mixed-effects models to account for individual differences

**For Loading Scenario Parameters:**
- Use `scenario1_trial_params.json` to programmatically access parameter specifications in your analysis scripts
- The JSON file is loaded by data processing tools in `/data_processing_tools/`
- For Python usage, import the centralized config loader: `from config_loader import load_scenario_config, Constants`

**For Scenario Selection:**
- Filter participants based on this assignment matrix when analyzing specific scenarios
- Example: Scenario DLC-Mixed-0 was completed by 40 participants (IDs: 0, 2, 3, 4, 7, 8, 9, 10, 13, 14, 16, 17, 19, 20, 22, 23, 24, 26, 29, 30, 32, 34, 35, 36, 38, 39, 41, 42, 44, 47, 48, 50, 51, 53, 54, 55, 58)
- Traditional scenarios (DLC 12-13, MLC 6-7) have complete data from all 60 participants

## Experiment Guide Documents

This folder also contains participant instruction materials used before the experiment:

### experiment_guide.pdf (English Version)
Pre-experiment briefing document provided to participants, including:
- **VR Equipment Introduction**: HTC Vive Pro headset, Tobii eye tracker, Logitech G29 steering wheel and pedals
